## 1. ゲームコンセプト

### エレベーターピッチ

「逆転裁判」のコミカル・ドラマチックな世界観を、リアル友人とブラウザだけで体験できるパーティーゲーム。  
AIが作った事件を、検察・弁護に分かれてリアルタイムで裁く。  
6人のAI陪審員はそれぞれ個性があり、証拠と言葉で票を動かしていく。  
突然飛び出す「**異議アリ！**」、証言台で崩れる証人、そして最後に明かされる **真相**。

### プレイヤーに提供する体験

| 役割 | 体験 |
|------|------|
| 検察官・弁護人 | AI陪審員の個性を読みながら証拠と言葉で票を動かす快感 |
| 全員 | リアルタイムで変わる票数と「異議アリ！」のドラマチックな展開 |

### 参考タイトル

- **大逆転裁判2** — AI陪審員システム・リアルタイム票変動・陪審員の個性
- **逆転裁判シリーズ** — 世界観・テンポ・異議あり演出
- **Jackbox Party Pack** — 気軽さ・2人から遊べる設計

## 3. プレイヤー・役割

### 基本情報

| 項目 | 内容 |
|------|------|
| プレイ人数 | **2人〜**（検察官 + 弁護人の最小構成でOK） |
| 推奨人数 | 2〜4人 |
| 1ゲームの目安 | 25〜35分 |
| ロール選択 | 先着制・重複不可 |

### 役割一覧

#### 🔵 検察官（1人・人間プレイヤー）
- 被告の有罪を立証する
- 証拠を提出し、証人を証言台に呼べる
- 弁護側の発言・証拠に異議を申し立てられる
- AI陪審員に向けて証拠・発言で働きかける
- **勝利条件：評決で「有罪4票以上」**

#### 🟢 弁護人（1人・人間プレイヤー）
- 被告の無罪を主張する
- 検察側の証拠・証言を崩す
- 独自の証拠・証人で反論する
- AI陪審員の個性を読んで最適な戦略を取る
- **勝利条件：評決で「有罪3票以下」（= 無罪多数）**

#### ⚖️ AI裁判長「ニシキ裁判長」（自動）
- ゲームの進行を管理する
- 異議の採否をリアルタイムで判断する
- 証人キャラクターを演じる（全員分）
- フェーズを自動遷移する
- 時々ユニークなコメントで場を盛り上げる

#### 👥 AI陪審員（6人・自動）
- 全18タイプから毎試合ランダムに6人を選出
- 全員の票（有罪/無罪）と現在の理由がリアルタイムで全員に見える
- 証拠提出・発言・異議の結果によって票が動く
- 票が変わるとき一言コメントを発する（大逆転裁判2風）
- **評決：最終フェーズで全員の票を集計**

### 勝利条件

| 役割 | 勝利条件 |
|------|---------|
| 検察官 | 最終評決で有罪 **4票以上** |
| 弁護人 | 最終評決で有罪 **3票以下**（無罪多数） |

> 同数（3-3）は「無罪」（疑わしきは罰せず）→ 弁護人の勝利。

## 4. AI陪審員システム

### 概要

大逆転裁判2の陪審員システムを参考にした、本ゲームの核心メカニクス。  
6人のAI陪審員は常に有罪/無罪の票を表示しており、試合中リアルタイムで変動する。  
各陪審員は個性（タイプ）を持ち、それぞれ異なる説得方法を持つ。

```
【法廷画面の陪審員エリア（常時表示）】

田中（感情屋）    工藤（論理派）    佐藤（頑固者）
  [無罪]            [有罪]            [有罪]
「被告が可哀想で」 「証拠不十分」   「絶対クロだ」

鈴木（流され屋）  渡辺（疑い深い）  中村（正義マン）
  [有罪]            [有罪]            [無罪]
「さっきの証拠は」 「目が怪しい」   「正義のために」

現在のスコア：有罪 4 — 無罪 2
```

### 票が動くタイミング

| トリガー | 内容 |
|---------|------|
| **証拠提出** | 証拠の強度・タイプに応じて関連する陪審員の票が動く |
| **証人の矛盾発覚** | 証言が崩れると「疑い深い」「論理派」などが反応 |
| **異議採用** | 異議を通した側に有利な票変動 |
| **発言内容** | 冒頭陳述・最終弁論のテキストをAIが評価 |
| **陪審員への直接説得** | 対象陪審員を選んで証拠+理由で説得できる（1ターン1回） |

### 陪審員への直接説得アクション

証拠・尋問フェーズで通常の行動の代わりに使用できる。

```
1. 説得したい陪審員を選択
2. 使用する証拠を選択（公開済み証拠から）
3. 理由を入力（「この証拠はXXXを示しており、あなたの懸念するYYYを解消します」）
4. AI（対象陪審員）が評価 → 票が動くか否か + リアクション
```

- 説得成功 → 票が翻る + 「連鎖反応」で隣の陪審員も影響を受けることがある
- 説得失敗 → 票が固定されてしばらく動かなくなる（逆効果のリスク）

## 5. ゲームフロー

### 全体フロー

```
ロビー（部屋作成・参加・役割選択）
  ↓
AI事件生成（約10秒）＋ AI陪審員6人をランダム選出
  ↓
開廷・冒頭陳述（各60秒）→ 陪審員の初期反応
  ↓
証拠・尋問フェーズ（最大15分）→ リアルタイム票変動
  ↓
最終弁論（各90秒）→ 流され屋・退屈な人への最後の訴え
  ↓
最終評決（自動集計）→ 票数確定
  ↓
評決発表（ドラマチック演出）
  ↓
真相開示（AI裁判長が発表）
  ↓
リザルト画面
```

### フェーズ1: ロビー

- ホストが部屋を作成 → 6文字の英数字コードが発行される
- 参加者はコードを入力して参加（スマホ可）
- 役割を先着順で選択（検察官・弁護人のみ。それ以外は観戦）
- 全員準備完了でホストが「開廷！」ボタン → 次フェーズへ

### フェーズ2: AI事件生成

- サーバーがClaude APIを呼び出してシナリオを生成
- 同時に18タイプからランダムで6人のAI陪審員を選出・初期票を設定
- 全員の画面に「事件資料を作成中...」と「陪審員を召喚中...」のローディング演出
- 生成されたシナリオはサーバー管理。情報の見え方：

| 情報 | 検察官 | 弁護人 | AI裁判長 |
|------|-------|-------|---------|
| 事件概要 | ✅ | ✅ | ✅ |
| 被告情報 | ✅ | ✅ | ✅ |
| 検察側証拠 | ✅ | ❌ | ✅ |
| 弁護側証拠 | ❌ | ✅ | ✅ |
| 証人の隠し情報 | ❌ | ❌ | ✅ |
| **真相（guilty/not）** | ❌ | ❌ | ✅ |
| AI陪審員の種類・初期票 | ✅ | ✅ | ✅ |

### フェーズ3: 開廷・冒頭陳述

1. **AI裁判長の開廷宣言**（自動）
   > 「開廷します。本件、〇〇事件について審理を開始します。被告人、起立。」

2. **陪審員紹介**（自動）
   - 6人のAI陪審員が名前・タイプ・初期票と一緒に画面に登場
   - 各陪審員が一言コメント（「有罪に決まってる」「まだわかりません」など）

3. **検察官の冒頭陳述**（60秒）
   - テキスト入力エリアに主張を入力・送信
   - 送信後、AIが内容を評価して陪審員が反応（票が少し動くことがある）

4. **弁護人の冒頭陳述**（60秒）
   - 同様

### フェーズ4: 証拠・尋問フェーズ（最大15分）

**ターン制：検察→弁護の交互行動**
- 各側最大5ターン（合計10ターン）
- タイムリミット15分（超過で自動的に次フェーズへ）
- 1ターンでできること（1つ選択）：

| アクション | 内容 | 陪審員への影響 |
|-----------|------|-------------|
| **証拠提出** | 手持ちの証拠を公開証拠ボードに追加（不可逆） | 証拠の強度・タイプに応じて関連する陪審員が反応 |
| **証人招致** | AI証人を呼んで主尋問（〜5往復） | 証人の証言・矛盾に応じて陪審員が反応 |
| **反対尋問** | 相手ターンに呼ばれた証人を尋問 | 証言を崩すと大きく票が動く |
| **陪審員説得** | 特定の陪審員を選んで証拠+理由で説得 | 成功すると票が翻る（失敗するとロック） |

**「異議アリ！」は相手のターン中いつでも使用可（詳細は第6章）**

### フェーズ5: 最終弁論（各90秒）

- 検察官が最後の主張を入力・送信（90秒タイマー）
- 弁護人が最後の主張を入力・送信（90秒タイマー）
- このフェーズは異議アリ！不可
- 最終弁論の内容をAIが評価 → **流され屋・退屈な人・感情屋**などが最後に大きく動く可能性
- 終了後、AI裁判長が「以上で弁論終結です」と宣言

### フェーズ6: 最終評決・発表

1. 「評決を読み上げます...」のサスペンス演出（2〜3秒の間）
2. AI陪審員の最終票が一人ずつ表示される（大逆転裁判2風）
3. 集計：有罪票数 vs 無罪票数
4. 判決を大きく表示（**有罪 / 無罪**）
5. 同数（3-3）は「無罪」（疑わしきは罰せず）→ 弁護人の勝利
6. 各プレイヤーの勝敗が表示される

### フェーズ7: 真相開示

> AI裁判長「それでは...この事件の真相を明かします。」

- 被告が「本当に犯人だったか否か」をAIが発表
- AI裁判長が真相の説明コメントを生成
- 評決と真相が一致したか否かを表示（おまけ情報）

## 6. コアメカニクス詳細

### 6.1 異議アリ！システム

#### 基本ルール

| 項目 | ルール |
|------|-------|
| 使用可能タイミング | 相手のターン中（冒頭陳述・最終弁論は除く） |
| 使用制限 | 各陣営 3回/フェーズ（証拠・尋問フェーズのみ） |
| 制限時間 | 異議理由の入力に15秒 |

#### 発動フロー

```
検察 or 弁護が「異議アリ！」ボタン押下
  ↓
全員の画面に「異議アリ！」全画面演出（約2秒）
  ↓
相手の発言がストップ
  ↓
異議申立人が15秒以内に理由を選択 + 自由入力
  ↓
AI裁判長（ニシキ裁判長）が採否を判断（3〜5秒）
  ↓
「採用！」 or 「却下！」の演出
  ↓
ゲーム再開（採用の場合は相手の発言が無効化）
  ↓
陪審員が採否に反応して票が動く（タイプによって異なる）
```

#### 異議の種類（フォームで選択）

- 矛盾（証拠と証言が矛盾している）
- 関係なし（本件と無関係な証拠・発言）
- 誘導尋問（証人に答えを誘導している）
- 根拠なし（主張に証拠が伴っていない）
- その他（自由入力）

#### AI裁判長の採否判断

- 異議理由の論理的妥当性をAIが評価（スコア7/10以上で採用）
- コミカルなコメント付きで裁定
  - 採用例：「確かに！これは明らかな矛盾です。異議を採用します！」
  - 却下例：「...言いがかりです。却下します。発言を続けてください。」

#### 陪審員への影響

| イベント | 影響を受けやすいタイプ |
|---------|-------------------|
| 異議採用（検察側） | 論理派・疑い深い・正義マン |
| 異議採用（弁護側） | 感情屋・お人好し・元被疑者 |
| 異議却下 | 流され屋・天邪鬼 |

### 6.2 証拠システム

#### 証拠の種類

| 種類 | 説明 | 例 |
|------|------|---|
| 物証 | 事件に関連する物・データ | 凶器、監視カメラ映像 |
| 証言録 | 目撃者・関係者の証言記録 | 隣人の目撃談 |
| アリバイ | 被告の所在証明 | レシート、GPS記録 |
| 動機資料 | 犯行動機を示す資料 | 遺産関係書類、恨みの手紙 |

#### 証拠と陪審員タイプの相性

- **物証（strong）** → 論理派・元警官が大きく反応
- **アリバイ** → 疑い深い・陰謀論者が反応
- **動機資料** → 感情屋・正義マン・お人好しが反応
- **証言録** → 推理オタク・専門家気取りが反応

### 6.3 連鎖反応システム

大逆転裁判2の「評決合一ノ儀」に着想を得た、票変動の連鎖演出。

- ある陪審員の票が変わると、隣接する陪審員に影響が出ることがある
- 特に **流され屋** は他の陪審員の票変動につられやすい
- 連鎖が起きると全員の画面でアニメーション演出が走る
- 連鎖確率はランダム要素を含む（毎回異なる展開になる）

```json
    {
      "id": "D1",
      "name": "証拠名",
      "type": "...",
      "description": "...",
      "strength": "...",
      "affects_juror_types": ["感情屋", "お人好し"]
    }
  ]
}
```

### 7.2 AI陪審員応答プロンプト

証拠提出・発言・説得のたびにAPIを呼び出し、各陪審員のリアクションを生成する。

```
systemPrompt:
あなたは「{name}（{nick}）」という陪審員です。
キャラクター：{description}
現在の票：{currentVote}
現在の理由：「{currentReason}」
説得されやすい条件：{move}

以下の出来事が起きました：
{event}

以下のJSONのみで応答してください：
{
  "newVote": "有罪 | 無罪 | 変わらず",
  "reason": "新しい理由（20文字以内）",
  "comment": "キャラクターとして一言（30文字以内）",
  "reaction": "surprised | convinced | dismissive | confused"
}
```

### 7.3 証人尋問プロンプト

証人ごとにシステムプロンプトに以下を含める：
- キャラクター設定（名前・職業・性格）
- 事件との関係・証言内容（表向き）
- 隠している情報・weakness（どんな質問で崩れるか）

証言は一貫性を保ちつつ、weaknessをついた質問には動揺・矛盾した返答を生成させる。

### 7.4 異議採否プロンプト

以下の情報をAPIに渡す：
- 対象の発言 or 証拠の内容・異議の種類・異議の理由（プレイヤー入力）
- 現在の公開証拠一覧

採否をスコア（1〜10）で出力させ、7以上で採用。コメントも生成させる。

### 演出方針

| 演出 | 内容 |
|------|------|
| 陪審員紹介 | 試合開始時に6人がアニメーションで登場 |
| 票変動 | 陪審員のカードがフリップ + 一言コメント表示 |
| 連鎖反応 | 複数の陪審員が連続でフリップするアニメーション |
